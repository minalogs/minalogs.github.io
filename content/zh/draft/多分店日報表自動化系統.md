# 多分店日報表自動化系統 Daily Report System

## 1. 專案背景與目標
簡要介紹問題場景（如每月報表建立重工、資料無備份等）

## 2. 階段一：資料清理與流程釐清
- 統一日報表欄位命名規則
- 員工常見錯誤點紀錄
- 判斷哪些資料要進月報表、哪些不需要

## 3. 階段二：資料架構與自動化
- 自動建立當月理帳報表（若無則複製範本）
- 開關台按鈕設計，觸發資料搬移
- 時間戳記的版本備份架構

## 4. 階段三：資料視覺化與回報
- 報表格式優化
- 分店與日別數據彙整
- 成本與收入標記，利潤估算顯示

## 5. 問題處理與系統進化
- 曾遇員工誤刪、誤操作後的設計調整
- 加入 alert 與備份 PDF 等機制

## 6. 成效回顧與反思
- 表格錯誤率下降 80%
- 員工操作步驟減少 50%
- 自己學到：流程設計 ≠ 自動化，而是理解使用者習慣
